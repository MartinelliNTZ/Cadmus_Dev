[2.3.16.5][18/04/2026] REFATORAÇÃO: Movida responsabilidade de validação e atualização de toolbar de BasePlugin.on_finish_plugin() para PyQtSignalManager. Criado novo sinal plugin_finished em _PluginSignalHub para comunicação entre plugins e signal manager. BasePlugin agora apenas emite o sinal; PyQtSignalManager (novo método _on_plugin_finished) executa lógica de reset de main_action, salvamento de prefs e reconstrução de toolbar. Funcionamento idêntico, mas com separação de responsabilidades.
[2.3.16.6][18/04/2026] INVERSÃO DE FLUXO: Invertida ordem de reconstrução de toolbar. Agora toolbar é reconstruída quando plugin ABRE (em _on_plugin_instantiated), não quando fecha. BasePlugin.__init__() inicializa _plugin_signal_context vazio; BasePlugin.init() emite sinal após setup completo; _on_plugin_instantiated() reconstrói toolbar; _on_plugin_finished() apenas atualiza main_action sem reconstruir. Logs ultra-detalhados com prefixos [init], [on_finish_plugin], [_on_plugin_instantiated], [_on_plugin_finished].
[2.3.16.7][18/04/2026] REFATORAÇÃO GRANDE: Arquitetura de fluxo de dados redesenhada. ToolRegistry agora é singleton com ToolList como source of truth durante sessão. Preferences usadas APENAS na primeira instância (init), nunca consultadas novamente. ToolRegistry.update_tool_main_action() gerencia estado de main_action + persist em preferences. PyQtSignalManager delega lógica para ToolRegistry (não consulta preferences). MenuManager._refresh_tool_main_actions() requisita ToolList atualizada de ToolRegistry (não carrega de preferences). Fluxo: ToolRegistry init → ToolList → MenuManager → Toolbar. Abertura plugin: signal → ToolRegistry.update_tool_main_action() → ToolList atualizada + prefs salvas → MenuManager.reconstruct_toolbar() requisita ToolList atualizada. Logging detalhado em cada componente com prefixos [ToolRegistry.update_tool_main_action], [_on_plugin_instantiated], [_on_plugin_finished].
[2.3.16.8][18/04/2026] LIMPEZA DE CÓDIGO: Removidos logs excessivos (debug redundantes) mantendo apenas marcos importantes (info/warn). Simplificados métodos em PyQtSignalManager (reduzidas linhas desnecessárias), ToolRegistry.update_tool_main_action() (refatorado para ~50 linhas), MenuManager._refresh_tool_main_actions() e reconstruct_toolbar() (removidas logs internas), BasePlugin.init() e on_finish_plugin() (reduzidas duplicações), Preferences.set_value_for_all_tools() e delete_value_for_all_tools() (simplificados sem perder funcionalidade).
