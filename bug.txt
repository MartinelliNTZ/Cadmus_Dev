Bug 3 permanece parcialmente — a correção introduziu um novo gap de lógica
A correção do falso positivo foi:
pythonif fits_current and fits_next and not dual_angle_break:
    if delta_azimuth <= config.light_azimuth_threshold:
        should_break = False
    # else: deixa os próximos elif decidirem
O problema é que o else não seta should_break = True nem is_central_outlier = True. O código cai fora do if/elif inteiro com should_break=False e is_central_outlier=False, e o ponto é absorvido como committed silenciosamente — o comportamento é idêntico ao falso positivo original para o caso delta_azimuth > light_threshold.
A intenção era "deixar os próximos elif decidirem", mas os elif são mutuamente exclusivos com o if fits_current and fits_next — quando entra no primeiro if, nenhum elif é avaliado. O ponto escapa sem decisão.
O que deveria acontecer: quando fits_current and fits_next mas delta_azimuth > light_threshold, o ponto deveria ser encaminhado ao bloco de confirmação prospectiva (o mesmo do elif past_stable and not fits_current), não simplesmente absorvido.
A correção limpa seria extrair a lógica de confirmação prospectiva em um método separado e chamá-la tanto do branch elif past_stable quanto do else do falso positivo com delta alto.

Bug de estado remanescente — mean_az usa raw_now como fallback quando az_history está vazia
pythonmean_az = MathUtils.axial_mean(recent_az) if recent_az else raw_now
No início de cada nova strip (logo após uma quebra), az_history está vazia. O mean_az vira raw_now (o azimute do ponto atual), fazendo delta_azimuth = axial_diff(raw_now, raw_now) = 0. Os primeiros 1–2 pontos de qualquer strip nova sempre terão score zero, independente de qualquer coisa. Isso é aceitável na maioria dos casos, mas pode suprimir uma quebra imediata se dois trechos muito curtos se sucederem rapidamente — os primeiros pontos do trecho 2 nunca disparam quebra porque o mean_az ancora neles mesmos.
Uma alternativa mais robusta seria inicializar az_history com o azimute do próprio ponto de quebra antes de limpar, ou usar o mean_az da strip anterior como prior fraco nos primeiros pontos da nova.
